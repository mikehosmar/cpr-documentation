Localization is a method of tracking the robot's location within an existing map. It is generally done using
laser scan data from a 2D LIDAR, and the robot's odometry. For this demo we are using the
[Nav2](https://github.com/ros-planning/navigation2) package launch localization.

See the sample [`robot.yaml`](configuration.mdx) for configuration details.

:::tip

Make sure you have installed the simulator before starting this tutorial. Take a look at the
[installation](../simulator/install.mdx) guide for details.

:::

## Launching Localization

:::note

When using the simulation, it is important to add `use_sim_time:=true` to the Nav2, SLAM, and
Localization launch files. When using a physical robot, either omit `use_sim_time` or set it
to `false`.

:::

**1.** [Start the simulation and Nav2](nav2.mdx#launching-the-simulation-and-nav2)
by following steps 1-3 of the Nav2 startup

**2.** Open a terminal and run
```bash
ros2 launch clearpath_nav2_demos localization.launch.py use_sim_time:=true
```

The default map used by `localization.launch.py` is a
[map](https://github.com/clearpathrobotics/clearpath_nav2_demos/blob/jazzy/maps/warehouse.yaml) of the simulated warehouse world.
If you are using a custom map, pass it in with the `map` launch argument:
```bash
ros2 launch clearpath_nav2_demos localization.launch.py use_sim_time:=true map:=/path/to/my/map.yaml
```

**3.** [Start Rviz](nav2.mdx#launching-the-simulation-and-nav2) and set the robot's initial pose
estimate by following steps 5-6 of the Nav2 startup

**4.** Give the robot a navigation goal

Use the **Nav2 Goal** tool in RViz to set a navigation goal pose. Click on the "Nav2 Goal"
button on the toolbar to select the tool, then click and drag on the map in Rviz to set the
destination location and orientation.

You will see a red path line appear, indicating the robot's planned path, and the robot will
start to drive along this path.
